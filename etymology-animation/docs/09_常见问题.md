# 常见问题

## Q1：AI 生成的字形变形了，笔画不对怎么办？

**原因**：运动幅度或创意度太高。

**解决**：
1. 降低运动幅度到 30-40%
2. 降低创意度到 0.5
3. 使用首尾帧模式（可灵），让字形由你提供的图片决定
4. 在 Prompt 中强调"保持字形完整不变形"
5. 多生成几次，挑最好的

## Q2：过渡动画太生硬，不像平滑变形？

**原因**：运动幅度太低，或两张图差异太大。

**解决**：
1. 适当提高运动幅度到 50%
2. 确保两张关键帧图片的字形在画面中位置一致
3. 如果两张图差异太大（如甲骨文和楷书），中间加一个过渡阶段

## Q3：背景闪烁或颜色不一致？

**原因**：关键帧图片的背景颜色不统一。

**解决**：
1. 所有图片统一为纯白背景（#FFFFFF）
2. 或者统一为透明背景（PNG）
3. 不要混用白底和黑底

## Q4：GIF 文件太大，网页加载慢？

**解决**：
1. 缩小 GIF 尺寸到 320×320 或 480×480
2. 降低帧率到 12fps（GIF 够用）
3. 用 ezgif.com 的"优化 GIF"功能压缩
4. 或者改用 MP4 + 自动播放（文件更小）

## Q5：找不到某个字的甲骨文怎么办？

**解决**：
1. 检查汉典的"字源"页面，有些字只有金文没有甲骨文
2. 用金文作为起点，少做一段
3. 如果连金文都没有，说明这个字可能是后起字（形声字居多），不适合做字源动画
4. 可以查"说文解字"的小篆字形作为最早的形态

## Q6：一个字要做几段动画？太多段太麻烦。

**建议**：
- 象形字（日、月、山等）：1 段即可（甲骨文直接到楷书）
- 会意字（取、采、休等）：2-3 段（有明显阶段性变化）
- 形声字：不建议做（动画价值低，部首相同的故事重复）
- 目标：每个字 3-6 秒总时长，不超过 3 段
